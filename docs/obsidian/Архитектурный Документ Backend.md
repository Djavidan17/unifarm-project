
**1. Общая роль backend**

Backend — это центральный мозг платформы.

Он выполняет четыре главные функции:

1. хранит данные
2. проверяет правила системы
3. обрабатывает действия пользователей
4. отдаёт данные frontend-у через API

Frontend **никогда не взаимодействует с базой напрямую**.

Все операции проходят через backend.

---

**2. Основные модули backend**

Чтобы код не превратился в хаос, backend делится на **логические модули**.

**2.1**. **Модуль пользователей (Users)**

Отвечает за всех участников системы.

Функции:

Регистрация пользователя

Вход в систему

Управление профилем

Назначение ролей

**Основные данные пользователя**

User

- id
- name
- email
- password_hash
- role
- phone
- created_at

**Роли**

В системе на старте будет три роли:

Farmer  
Buyer  
Admin

Это позволит ограничивать доступ к функциям.

Например:

- только фермер может размещать товар
- только админ может удалять пользователей

---

**2.2. Модуль товаров (Products)**

Этот модуль — основа торговой площадки.

Фермеры смогут размещать свои товары.

**Основные функции**

Создание товара

Редактирование товара

Удаление товара

Просмотр списка товаров

Поиск товаров

**Структура данных товара**

Product

- id
- title
- description
- price
- quantity
- category
- owner_id
- created_at

owner_id связывает товар с пользователем-фермером.

---

**2.3. Модуль сделок (Deals)**

Этот модуль отвечает за взаимодействие между покупателем и фермером.

**Как будет работать сделка**

1. покупатель находит товар
2. отправляет запрос на покупку
3. фермер подтверждает или отклоняет
4. сделка получает статус

**Структура сделки**

Deal

- id
- product_id
- buyer_id
- farmer_id
- status
- created_at

**Возможные статусы**

pending  
approved  
rejected  
completed

---

**2.4. Модуль информации (Info)**

Это информационный раздел платформы.

Он содержит:

новости  
объявления  
полезные статьи

**Структура**

Article

- id
- title
- content
- author
- created_at

**3. API взаимодействие**

Frontend не работает напрямую с кодом backend.

Он общается через **REST API**.

Это обычные HTTP-запросы.

Примеры.

Получить список товаров

GET

/api/products

Создать товар

POST

/api/products

Получить профиль пользователя

GET

/api/users/me

Создать сделку

POST

/api/deals

Ответ backend всегда возвращается в формате **JSON**.

Пример ответа:

{  
  "id": 21,  
  "title": "Tomatoes",  
  "price": 3.5,  
  "quantity": 100  
}

---

**4. Авторизация**

Пользователь после входа получает **JWT-токен**.

Этот токен используется во всех запросах.

Пример:

Authorization: Bearer TOKEN

Backend проверяет токен и понимает:

кто пользователь  
какая у него роль
